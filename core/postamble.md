# Postamble

Use this checklist before returning a final research output.

## 1. Completeness

- The final answer contains the complete Markdown report requested by the route.
- Required sections and tables are present.
- Important missing data is stated plainly.
- The output is not just a short summary unless the user explicitly asked for one.

## 2. Evidence Check

- Facts are separated from market consensus and reasoning.
- Weak or conflicting sources are identified where they affect the conclusion.
- No unavailable data is invented.

## 3. Compliance Check

- The report does not claim to be personalized financial advice.
- Buy/sell language is framed as assumptions, scenarios, risk-return profile, and verification points.
- Ratings, target prices, and analyst estimates are treated as market views, not facts.

## 4. Final Response Shape

For the public edition, return the complete Markdown in the current conversation. Local file output and external service upload are optional user-specific extensions, not completion requirements.
