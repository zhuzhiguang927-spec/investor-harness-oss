# Q&A 双链复利

> 知识复利的底层机制。daily feed 解决"信息怎么流入"，question list + Q&A 双链解决"研究怎么沉淀"。

## 为什么需要

投研不只是被动接收信息——你要主动提问题、做分析、准备会议、跟进待办。这些任务**需要被跟踪、有上下文、进度可见**。

## Question List 机制（见分析师 / 管理层之前）

要见一家公司的分析师 / 管理层之前：

1. 建一个 question list 文件，列出问题
2. `librarian` 对 vault 做全量扫描
3. 在每个问题下面**自动附上一段"vault 扫描初步结论"**——交叉综合现有的所有纪要、研报、公众号、模型数据，列出当前知识库已经能回答的部分、已知的矛盾点、以及建议的追问方向

打印出来就是一份完整的会前 briefing。

### 副作用：会前自动纠偏

这套机制最大的副作用：**vault 扫描经常会主动纠正你的认知偏差**——你以为的数字和 vault 里实际能找到的数字经常有出入。系统把两个数字摆在一起，矛盾立刻暴露——这种纠正如果在会上才发生，代价就大了。

## 会后归档（Q&A 双链触发）

会后 question list 转化为 Q&A 存档：

1. 逐题回填分析师回答
2. 标注来源（visiting record + 时间戳）和日期
3. 没问到的、回避的、和 vault 信息矛盾的分别标状态
4. 然后**触发 wiki 级联更新**——新数据更新对应段落、Key Take-Away 刷新、催化剂日历加入分析师提到的时间节点（港股通纳入、产能投产）、Next steps 从"还缺什么"自动生成

完整链路：**问之前知道什么 → 问了什么 → 得到了什么 → 还缺什么**，全部可追溯。

## Q&A 双链机制

两条链合在一起就是**复利**。

### 链 1：问题 → 答案 → 存档

每次问了一个问题，不管是 vault 内检索、NLM 跨源综合、还是分析师口头确认，答案都**持久化存储**：

- 带**路由方式**（哪种工具拿到的）
- 带**来源标注**（wikilink 到原始文件）
- 带**置信度评分**（A 级 / B 级）

**高置信度的答案**下次直接复用。
**低置信度的答案**留着做质量追踪，等更好的来源出现时覆盖更新。

### 链 2：答案 → wiki → 下次提问的上下文

答案不只是存在 Q&A 存档里，它**还会更新 wiki page 对应段落**。下一次 vault 扫描的时候，更新后的 wiki 内容会成为新问题的背景知识——question list 的"vault 扫描初步结论"自动包含了上次的研究成果。

## 复利公式

```
新建 coverage
   → 原始材料入库
   → wiki v1
   → 分析师会议
   → question list 引用 wiki v1
   → 获得新信息
   → Q&A 归档
   → wiki v2（吸收了分析师信息）
   → 下次季报
   → question list 引用 wiki v2
   → 更精准的问题
   → 新的 Q&A
   → wiki v3 ...
```

每一轮循环：

- wiki **变厚一层**
- question list 的**起点变高一层**
- 提出的问题**更精准**
- 获得的答案**更有价值**

三个季度下来，一家公司的 wiki page 不再是"材料汇总"，而是一份**经过反复交叉验证、带完整研究脉络的知识资产**。

## 复利是可观测的

§4 Key Take-Away 的 🔬 研究桶清楚地列着每一次研究活动——什么时候做了哪个分析，什么时候准备了哪个会议，什么时候回填了哪些答案。**整个研究历程一目了然**。
