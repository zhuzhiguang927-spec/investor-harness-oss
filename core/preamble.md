# Preamble

Run this lightweight preflight before company, industry, comparison, event-driven, or other research outputs.

## 1. Identify The Task

Determine:

- target company, industry, theme, or event
- market scope: `CN-A`, `HK`, `US`, `GLOBAL`, or mixed
- intended route / skill
- user focus and time horizon

## 2. Check Source Requirements

For company or industry research, read and synthesize recent five-year broker / sell-side reports when available before qualitative conclusions.

Use this public source order:

1. broker / sell-side report text or PDF
2. official disclosures and financial reports
3. structured market and financial data
4. Chinese web search and broader web search
5. user-provided materials

If full report text cannot be found, state the gap briefly when it affects the conclusion.

## 3. Check Existing Context

If the current workspace has prior notes, reports, or user-provided materials relevant to the task, read them before writing. If no prior context exists, treat the task as first-pass research.

## 4. Evidence Discipline

Separate:

- `公开事实`
- `财报披露`
- `市场共识`
- `合理推演`
- `待核验假设`

Do not invent unavailable data. Use `not disclosed`, `not found`, or `not comparable` when appropriate.

## 5. Output Contract

For the public edition, the default deliverable is a complete Markdown report in the current conversation. Local file output and external service upload are optional user-specific extensions, not completion requirements.

## 6. Compliance

Do not present the output as personalized financial advice. When the user asks "should I buy", answer in terms of assumptions, evidence, risk-return profile, and verification points rather than unconditional buy/sell instructions.
