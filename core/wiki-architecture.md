# Wiki Architecture — 14 段标准 wiki page

> Librarian 升级的核心载体。每家被覆盖的公司都有一份 wiki page，结构是固定的 14 段。

## 为什么是 14 段

Holdings（已建仓的持仓）和 Coverage（在跟踪、还没建仓的标的）合起来就是 focus list——所有这些公司的 wiki page 遵守**同一套 14 段标准结构**，目的是让任何一家公司打开就能直接读，结构记忆零成本。

## 14 段结构

| § | 段落 | 更新频率 | 谁更新 |
|---|---|---|---|
| §1 | Snapshot（基本信息 / 业务概览 / 股东） | 季度 / 重大事件 | 人工 |
| §2 | Model Snapshot（财务模型关键数）| 模型更新触发（registry） | librarian 自动 diff |
| §3 | **Valuations（估值，实时）** | **每天** | librarian 每天拉股价，按 当天股价/模型基准股价 比例缩放 EV/PE/EV-EBITDA |
| §4 | **Key Take-Away（关键速记）** | 每天滚动聚合 | librarian 扫 vault 近 90 天文件，按 7 桶分类 |
| §5 | 关键指引 Key Guidance | 业绩会 / 调研触发 | 人工录入，标 B 级 |
| §6 | Known Truth（高密度沉淀） | 低频，每次都是认知质变 | 人工录入 |
| §7 | Analyst Ratings（卖方覆盖） | 每周 | librarian 从券商索引自动汇总 |
| §8 | 近期 Thesis（核心论点） | 月度 / 重大事件 | 人工录入 |
| §9 | 催化剂日历 | 滚动 | librarian 从 Q&A "Next steps" 自动生成 |
| §10 | Q&A 存档 | 每次分析师会议 / 重要问答 | librarian 写入，人工标置信度 |
| §11 | 调研纪要索引 | 每次 visiting record 落盘 | librarian 自动 |
| §12 | 关键风险与退出条件 | 月度 | 人工 |
| §13 | Diff Log（变更追溯） | 每次 wiki 段落改动 | librarian 自动 |
| §14 | 仍需补的资料 / 待研究队列 | 滚动 | 自动 + 人工补充 |

## 高密度沉淀资产（§5 / §6 / §8）

更新频率不高，但每个字都来自反复验证和判断。是这家公司**真正的高质量信息**：

- **关键指引（§5）**：管理层在电话会、业绩交流、调研里给出的指引数字——全年销量目标、毛利率区间、CapEx 节奏、海外扩张时点、利润率改善路径。指引本身属于 B 级（管理层口径有 talk book 风险），但价值在于它锚定了未来——后续每个季度实际数字都要和这里对照，看 deliver 还是 miss，miss 的方向和幅度往往就是 thesis 改不改的触发点。
- **Known Truth（§6）**：经过多源交叉验证、反复确认的稳定事实——商业模式的核心机制、关键利润驱动因素、市占率结构、护城河来源、行业地位。这一段更新频率最低，但每次更新都意味着对公司的认知发生了质变（比如"原来以为的护城河其实在被削弱"），所以每次改动都值得追溯到具体触发证据。
- **近期 Thesis（§8）**：当前对这家公司的核心论点——为什么 long/short/hold、市场定价错在哪里、催化剂是什么、风险在哪里、退出条件是什么。**这是整个研究的最终落点**。其他所有段落（估值、Key Take-Away、关键指引、Known Truth）都是为这一段服务的——它们要么是 thesis 的证据，要么是 thesis 的反证。

## 实时估值（§3 Valuations）

`librarian` 每天拉最新股价，按 **当天股价 / 模型基准股价** 的比例对模型里的市值、EV、PE、EV/EBITDA 这些估值指标整体缩放一遍。

- 模型本身**季度更新一次**（基本面有没有变化、模型该不该改，需要人判断）
- wiki 上的估值**每天都是活的**（"今天股价多少、对应估值多少"是每天都要回答的问题，必须自动化）

## 模型数据 registry

每个数据点注册在 registry 里，记录了从哪个 Excel 模型的哪个 tab 的哪一行提取。组合内重点持仓单家公司大致注册 **30+ 数据点**。

`librarian` 定期扫描模型文件的修改时间。一旦发现变化，自动提取全部注册数据点和上次快照做 diff，变化写进日志（§13 Diff Log）。**模型更新后 wiki 上不会悄悄过期**——diff 会明确告诉你哪些数字变了、变了多少。

## 写作约束

- wiki page **不是**外发研报。是内部知识资产。
- 每个数字来源（§13 之外）必须能追溯到一行 wikilink → 原始来源
- §5 / §10 的关键数字和结论必须有来源或口径备查，但正文不堆标签化来源分级
- §6 Known Truth 改动必须在 §13 留 diff（包含触发证据）
