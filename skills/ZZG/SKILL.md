### Step 2: 拉取市场数据和财务数据

**职责：** 通过 data_agent.py 拉取全部市场数据 + 财务数据

**执行步骤：**

1. **调用 data_agent.py**（akshare雪球行情/同花顺预测 + akshare年报/季报 + 东方财富F10产品/区域/毛利率）：
   ```
   python {HARNESS_ROOT}\skills\ZZG\data_agent.py {code} {公司名} --out "{workspace}/{code}_data.md"
   ```
   脚本自动获取5个模块数据并输出MD文件。市场实时指标走 akshare 雪球个股行情，市场预测初值走 akshare 同花顺盈利预测；年报和季报也走 akshare。
   - **产品/区域/毛利率数据**：自动从东方财富F10 BusinessAnalysis API获取（结构化JSON，无需手动搜索）
   - 如果东方财富API无数据（极端情况），可通过 `--search-data` 传入手动搜索的JSON
   - **市场预测备注**：AkShare 能找到几年的预测收入和预测利润，就在市场表中填几年的数据；如果只找到2026E，或只找到2026E/2027E，不要为了凑齐年份伪造2027E/2028E。
   - **2026E强制要求**：2026E预测收入和2026E预测利润必须拿到，因为 `Forward PE` 和 `Forward PS` 要基于2026E计算。若 AkShare 连这两个2026E字段都拿不到，必须联网搜索券商研报或盈利预测资料补齐2026E收入与归母净利润，再重算前瞻估值。

2. **券商盈利预测聚合（强制，最终报告必须覆盖 data_agent 初始预测）**：
   - 搜索近2个月内发布的所有可找到的券商预测，优先查券商研报原文或研报平台披露的预测表，关键词至少覆盖：
     - `{公司名} 研报 2026E 2027E 2028E 营收 净利润`
     - `{公司名} 盈利预测 2026 2027 2028 券商`
     - `{公司名} 年报点评 2026E 2027E 2028E`
   - 对同一券商同一份报告去重；同一券商近2个月有多份报告时，保留最新预测，避免重复加权。
   - 内部整理每家券商的报告日期、券商名、2026E/2027E/2028E收入与归母净利润，用于去重和计算平均值。
   - 用可用样本的算术平均值作为最终报告市场数据中的 `2026E/2027E/2028E预测收入` 和 `2026E/2027E/2028E预测利润`；`Forward PE/PS` 基于该均值重算。
   - 最终报告只输出算出的预测均值，不输出每家券商预测、报告日期、样本表、样本数量或预测区间。
   - 如果近2个月没有券商预测，改用当前能找到的最新券商预测，多找几家不同券商；目标至少3家。确实不足3家时内部按可得样本计算，并且不要伪造一致预期。
   - 同花顺或 data_agent 返回的预测只能作初始线索，不能替代这一步人工聚合。

3. **联网搜索补充**（仅在东方财富API无数据时需要，通常不需要）：
   - **搜索引擎优先级**：搜狗微信搜索（公众号文章）→ 头条搜索 → 东方财富搜索 → 内置web_search
   - 搜狗微信搜索示例：`web_fetch(url="https://weixin.sogou.com/weixin?type=2&query={公司名}+2024年报+分产品+营收+占比&page=1")`
   - 将搜索结果整理为JSON格式，写入 `{workspace}/{code}_search.json`
   - 重新运行：`python data_agent.py {code} {公司名} --out "{workspace}/{code}_data.md" --search-data "{workspace}/{code}_search.json"`
